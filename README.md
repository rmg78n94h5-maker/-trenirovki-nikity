# Тренировки Никиты — PWA 1.5.2

Персональное офлайн-приложение для тренировок на судне. Проект не требует App Store, Apple Developer и серверной базы данных.

## Что уже заложено

- Профиль: Никита, 34 года, 173 см, 77 кг.
- Цели: убрать живот и бока, укрепить нижнюю часть пресса, сделать тело плотнее и эстетичнее без сильного набора массы.
- Оборудование: степпер без поручня, регулируемая штанга, разборные гантели, ролик для пресса, стул, коврик, собственный вес и доступный мультитренажёр.
- Основная программа: 6 тренировочных дней + восстановительный день.
- Дополнительная программа: сохранённый цикл журнала «Тело в форме 2.0» на 4 дня.
- Стартовые цели питания: 2700 ккал в тренировочный день, 2500 ккал в восстановительный, белок 160 г. Всё редактируется.
- Точные прошлые рабочие веса не были явно зафиксированы в доступных страницах журнала, поэтому приложение их не выдумывает. После первой сохранённой тренировки оно показывает прошлый вес и предлагает прогрессию.

## Возможности текущей версии

- Главный экран с умным стартом: продолжить цикл, повторить прошлую тренировку или выбрать другой день без сдвига цикла.
- Контроль боли перед тренировкой: можно выбрать область, степень 1–10 и комментарий.
- В списке зон есть «Паховая область / низ живота» для осторожной тренировки при дискомфорте в этой зоне.
- Рискованные упражнения помечаются предупреждением ⚠️ и предлагают снизить вес, заменить, пропустить или оставить как есть.
- Боль можно отметить прямо возле упражнения; отдельная история боли хранится локально по профилю.
- Черновик тренировки показывается отдельно и защищает данные от случайной перезаписи.
- Полное прохождение тренировки с весами, повторениями, сложностью и комментариями.
- Крупные кнопки −/＋ для веса, повторов и длительности; число по-прежнему можно ввести вручную.
- Короткая офлайн-заставка при запуске без внешних файлов и без задержки после готовности приложения.
- В разделе «План» переключатель программ очищен от повторов: стандартные программы не размножаются в верхней ленте.
- Добавлена кнопка «＋ Создать» и конструктор программы: выбираешь 1–7 тренировочных дней, затем собираешь каждый день через редактор.
- При новом выполнении упражнения автоматически подставляются значения завершённых подходов из последней тренировки.
- Таймер отдыха, звуковой сигнал и попытка вибрации.
- Короткий режим «Нет сил» на 15–20 минут, который продолжает текущий цикл.
- Замена и пропуск упражнений.
- Автоматические предложения по прогрессии без принудительного изменения веса.
- История тренировок и общая нагрузка.
- Замеры тела и простые SVG-графики.
- Отслеживание рабочих весов и времени степпера.
- Локальные фотографии прогресса и сравнение двух дат.
- Редактор дней, упражнений, порядка, подходов, повторений, веса и отдыха.
- Несколько программ и создание своих.
- Экспорт/импорт JSON, включая отдельный полный экспорт с фото.
- IndexedDB для данных и Service Worker для офлайн-режима.
- IndexedDB v3: добавлено отдельное локальное хранилище истории боли.

## Как проверить на компьютере

Service Worker не работает при открытии `index.html` как обычного файла. Запусти локальный сервер:

```bash
python3 -m http.server 8080
```

Затем открой `http://localhost:8080`.

## Бесплатное размещение на GitHub Pages

1. Создай бесплатный аккаунт на GitHub.
2. Создай новый **public**-репозиторий, например `nikita-workouts`.
3. Распакуй архив и загрузи **содержимое папки** `nikita-workouts-pwa` в корень репозитория.
4. Открой **Settings → Pages**.
5. В разделе **Build and deployment** выбери `Deploy from a branch`.
6. Выбери ветку `main` и папку `/ (root)`, затем `Save`.
7. Подожди несколько минут. GitHub покажет HTTPS-адрес вида:
   `https://ИМЯ.github.io/nikita-workouts/`
8. Открой этот адрес в Safari на iPhone при наличии интернета.

Важно: исходный код в public-репозитории виден всем, но в проекте нет твоих фотографий, истории или замеров. Они появляются только локально на телефоне после запуска.

Официальная справка GitHub Pages:
https://docs.github.com/pages/getting-started-with-github-pages/creating-a-github-pages-site

## Как добавить на экран iPhone

1. Открой опубликованный HTTPS-адрес именно в Safari.
2. Нажми кнопку **Поделиться**.
3. Выбери **На экран «Домой»**.
4. Оставь название «Тренировки Никиты» и нажми **Добавить**.
5. Запусти приложение с иконки, пока интернет ещё есть, чтобы Service Worker закэшировал файлы.
6. После этого основные экраны и сохранённые данные работают без постоянного интернета.

## Резервные копии

В приложении открой **Ещё → Резервная копия**.

- **Данные JSON** — программа, история, подходы, замеры и настройки без фотографий.
- **С фото** — полный JSON, фотографии кодируются внутрь файла; копия может быть большой.
- **Импортировать копию** — выбирает JSON из приложения «Файлы» и полностью восстанавливает базу. Перед импортом копии без фото приложение предупреждает, если локальные фотографии будут удалены.

Практический режим: экспортировать обычный JSON раз в неделю и полный файл с фото раз в месяц или после важных фотографий.

## Фото и приватность

Фото хранятся в IndexedDB внутри origin опубликованного сайта. Они не отправляются на сервер приложением. Но локальное хранилище PWA нельзя считать единственной копией: данные могут исчезнуть после удаления иконки, очистки Safari, смены домена или критической нехватки места.

Безопасная схема:

- оригиналы оставить в «Фото» или закрытой папке «Файлы»;
- в PWA использовать копии для сравнения;
- периодически делать полный экспорт.

Никогда не добавляй личные фотографии прямо в папку GitHub Pages — тогда они станут публичными.

## Обновление приложения

Версия приложения задаётся один раз в `version.js`. Service Worker использует её для нового имени кэша автоматически. Текущий кэш: `nikita-workouts-v1.5.2-update-manager-fix`.

В v1.4.0 IndexedDB повышена до v3: добавлено хранилище `painEntries` для истории боли. В v1.5.0 миграции базы нет: рекорды считаются по истории тренировок и сохраняются внутри завершённой тренировки. В v1.5.1–v1.5.2 база также не менялась: добавлен и уточнён менеджер обновления PWA-кэша. Существующие профили, тренировки, программы, замеры и фотографии не очищаются.

Загрузи обновлённые файлы в GitHub. При следующем запуске онлайн приложение получит новую оболочку. Локальная IndexedDB с историей не стирается.

## Ограничения iPhone

- Вибрация через Web API на iPhone может не сработать; предусмотрен звуковой сигнал.
- Фоновые таймеры iOS может приостанавливать, но после возвращения приложение пересчитывает остаток по реальному времени.
- Локальные данные привязаны к точному адресу сайта. Переезд на другой домен требует экспорта и импорта.
- Для Service Worker нужен HTTPS; GitHub Pages его предоставляет.


## v1.5.0 — Рекорды

Добавлен модуль полезных рекордов:

- новый рекорд после завершения тренировки показывается отдельным окном;
- в истории тренировки отображается количество рекордов и подробности;
- в разделе **Прогресс → Рекорды** есть последние рекорды и лучшие текущие показатели;
- считаются максимальный вес, лучший результат «вес × повторы», больше повторов с тем же весом, лучший объём, рекорд без отказа, лучшие временные упражнения и степпер;
- если рекорд получился на фоне сильной боли, приложение сохраняет результат, но предупреждает не повышать нагрузку через боль.

IndexedDB остаётся v3. Новые хранилища не добавлялись.

## v1.5.1 — Обновление приложения

Технический патч для iPhone/PWA:

- Service Worker регистрируется с версией в URL, чтобы iOS быстрее замечала новый файл;
- приложение проверяет `version.js` при запуске и периодически во время работы;
- при наличии новой версии появляется нижний баннер «Доступно обновление»;
- в разделе **Ещё → Обновление приложения** добавлены кнопки «Проверить обновление» и «Обновить кэш»;
- «Обновить кэш» удаляет только PWA-кэши `nikita-workouts-*`, не трогая IndexedDB;
- перед перезагрузкой текущий черновик тренировки сохраняется, если тренировка была в процессе.

IndexedDB остаётся v3. Новых хранилищ и миграций нет.


## v1.5.2 — Фикс обновления

Маленький технический патч после проверки v1.5.1 на iPhone:

- баннер обновления больше не должен исчезать сам через пару секунд;
- если версия уже стала актуальной, приложение показывает понятное сообщение об успешном обновлении;
- в разделе **Ещё → Обновление приложения** добавлены статусы: текущая версия, состояние кэша и последняя проверка;
- перед ручным «Обновить кэш» приложение отдельно предупреждает, что IndexedDB не очищается;
- скрипты и стили подключаются с параметром версии, чтобы Safari/PWA меньше держал старые файлы;
- Service Worker отдаёт JS/CSS/manifest по схеме network-first с fallback в кэш, чтобы свежий код подтягивался быстрее, но офлайн-режим сохранялся.

IndexedDB остаётся v3. Новых хранилищ и миграций нет.
