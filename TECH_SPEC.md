# Краткое техническое задание — «Тренировки Никиты»

## 1. Назначение

Личное PWA для iPhone, рассчитанное на судовой режим, слабый интернет и локальное хранение данных. Главный сценарий — открыть приложение с домашнего экрана, пройти тренировку, записать каждый подход и увидеть фактический прогресс.

## 2. Персональные исходные данные

- Никита, 34 года.
- Рост 173 см, текущая масса 77 кг.
- Цели: живот и бока, нижняя часть пресса, плотная эстетичная форма без сильного набора массы.
- Работа на ногах в камбузе.
- Инвентарь: степпер без поручня, штанга, гантели, ролик, стул, коврик, собственный вес, доступный мультитренажёр.
- Ограничения: качка, необходимость устойчивых замен, история операции на паховой грыже, отсутствие натуживания и максимальных весов.
- Старые точные рабочие веса не выдумываются; приложение получает их после первого сохранённого выполнения.

## 3. Основная программа

Основной активный цикл:

1. Верх тела + пресс.
2. Ягодицы + задняя цепь.
3. Пресс, бока + степпер.
4. Спина, бицепс + плечи.
5. Ноги + ягодицы.
6. Круговая тренировка.
7. Восстановление + степпер.

Также сохранён альтернативный цикл журнала «Тело в форме 2.0» на четыре тренировочных дня.

## 4. Экраны

- **Сегодня:** дата, план, упражнения, серия, недельная статистика, последний замер, питание, последняя тренировка.
- **Тренировка:** подходы, вес, повторения, сложность, прошлый вес, предложение нагрузки, таймер, замена, пропуск, комментарии.
- **План:** несколько программ, выбор активной, изменение текущего дня, редактор упражнений.
- **История:** день/неделя/месяц/всё, длительность, нагрузка, выполнение, детальный просмотр.
- **Прогресс:** вес, талия, тренировки, рабочие веса, степпер, фотографии.
- **Ещё:** профиль, калории/БЖУ, сигналы, экспорт/импорт, установка PWA.

## 5. Хранение

IndexedDB, базы/хранилища:

- `settings` — активная программа, текущий день, звук, вибрация.
- `profile` — физические данные, цели, ограничения.
- `nutrition` — калории и БЖУ.
- `exercises` — библиотека и пользовательские упражнения.
- `programs` — программы, дни и порядок упражнений.
- `workouts` — результаты подходов и комментарии.
- `measurements` — вес и объёмы.
- `photos` — локальные Blob-файлы фотографий.
- `meta` — версия базы и черновик незавершённой тренировки.

## 6. Прогрессия

- Все подходы лёгкие: предложить небольшой рост веса или +1–2 повтора.
- Нормально: повторить нагрузку.
- Тяжело: оставить нагрузку.
- Отказ/невыполнение: предложить уменьшение на 5–10%.
- Перерыв более 14 дней: предложить минус 10%.
- Перерыв более 30 дней: предложить минус 20%.
- Решение всегда подтверждает пользователь.

## 7. Офлайн

Service Worker кэширует оболочку приложения. IndexedDB хранит пользовательские данные. После первого успешного открытия по HTTPS приложение запускается без постоянного интернета.

## 8. Резервная копия

- Обычный JSON без фото.
- Полный JSON с фото в Data URL.
- Импорт с полной заменой локальной базы.

## 9. Стек

- HTML5.
- CSS без внешних библиотек.
- Vanilla JavaScript без сборки.
- IndexedDB.
- Service Worker.
- Web App Manifest.
- SVG-графики, генерируемые приложением.

Выбран стек без React и npm-зависимостей, чтобы архив было проще разместить, обновлять и кэшировать при слабом интернете.
