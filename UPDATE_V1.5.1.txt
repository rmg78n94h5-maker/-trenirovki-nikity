# Обновление v1.5.1 — Обновление приложения

База: свежий ZIP `-trenirovki-nikity-main 12.zip`.

Это маленькое техническое обновление по правилу патч-версии: 1.5.0 → 1.5.1.

Что добавлено:

1. Менеджер обновления PWA:
   - при запуске приложение проверяет свежую версию через `version.js`;
   - проверка повторяется примерно раз в 15 минут, пока приложение открыто;
   - если на сервере появилась новая версия, показывается нижний баннер «Доступно обновление».

2. Раздел «Ещё → Обновление приложения»:
   - отображается текущая версия приложения;
   - кнопка «Проверить обновление» вручную проверяет свежую версию;
   - кнопка «Обновить кэш» принудительно очищает только PWA-кэши приложения и перезагружает оболочку.

3. Service Worker:
   - регистрируется с параметром текущей версии в URL;
   - при установке и сетевом обновлении оболочки использует `cache: reload`, чтобы iOS меньше держала старые файлы;
   - умеет принимать команды `SKIP_WAITING` и `CLEAR_APP_CACHES`;
   - новый кэш: `nikita-workouts-v1.5.1-update-manager`.

Данные:

- IndexedDB остаётся v3.
- Новых хранилищ нет.
- Миграции базы нет.
- Профили, программы, история, замеры, фото, питание, боль, рекорды и черновики не очищаются.
- При ручном обновлении, если тренировка в процессе, черновик сохраняется перед перезагрузкой.

Важно:

Полностью отменить особенности iOS Safari нельзя, но теперь вместо многократного закрытия/открытия можно использовать кнопку «Обновить кэш».
