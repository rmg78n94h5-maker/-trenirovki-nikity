# Отчёт проверки v1.7.0

Проверено статически:
- node --check app.js — успешно;
- node --check db.js — успешно;
- node --check seed.js — успешно;
- node --check exercise-guides.js — успешно;
- manifest.webmanifest валиден JSON;
- version.js содержит 1.7.0;
- service-worker.js содержит SERVICE_WORKER_RELEASE 1.7.0;
- index.html подключает JS/CSS с параметром v=1.7.0;
- все файлы APP_SHELL из service-worker.js существуют;
- дублей function-объявлений в app.js не найдено;
- IndexedDB осталась v3;
- ZIP проверен после сборки.

Что не проверялось:
- полноценный интерактивный прогон на iPhone/PWA, потому что sandbox не воспроизводит iOS PWA и работу GitHub Pages.

Ручной тест после загрузки:
1. Открыть приложение с интернетом.
2. Проверить версию в «Ещё → Обновление приложения».
3. Открыть «Прогресс → Разгрузка».
4. Проверить карточку «Восстановление» на главной.
5. Если есть признаки усталости, стартовать тренировку и проверить баннер разгрузки.
6. Нажать «Применить −20% к этой тренировке» и убедиться, что веса/время незавершённых подходов снижены, а данные профиля не пропали.
