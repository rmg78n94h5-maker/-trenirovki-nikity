# Проверка v1.4.0 — Контроль боли

## Проверено статически

- `node --check` для всех JS-файлов: успешно.
- `manifest.webmanifest`: валидный JSON.
- Дубликатов функций в `app.js`: не найдено.
- Версия приложения: 1.4.0.
- Service Worker release: 1.4.0.
- Кэш: `nikita-workouts-v1.4.0-pain-control`.
- Все файлы APP_SHELL существуют.
- ZIP проверен на повреждения.

## Проверено по коду

- IndexedDB повышена до v3.
- Хранилище `painEntries` создаётся при миграции.
- `painEntries` экспортируется и импортируется в резервных копиях.
- Удаление профиля удаляет связанные записи боли.
- Удаление тренировки удаляет связанные записи боли.
- Перед стартом тренировки открывается проверка боли.
- При ответе «Нет, всё нормально» тренировка запускается без отметки боли.
- При выборе области и степени боли тренировка получает `preWorkoutPain`.
- Рискованные упражнения получают `painRisk` и предупреждение ⚠️.
- Зона «Паховая область / низ живота» учитывает упражнения с давлением на корпус/низ живота: приседы, тяги, выпады, ролик, планки, подъёмы ног и похожие движения.
- В упражнении можно добавить отдельную отметку боли.
- Действия предупреждения: снизить вес, заменить, пропустить, оставить.
- История боли выводится в разделе «Ещё».

## Ограничение проверки

Полный интерактивный прогон через Chromium внутри sandbox был заблокирован средой (`ERR_BLOCKED_BY_ADMINISTRATOR` при открытии localhost). Поэтому браузерный сценарий заявлен не как пройденный, а как статически проверенный по коду.
